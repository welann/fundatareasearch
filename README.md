# Fun Data Research

这个仓库用于沉淀一些有趣的数据分析页面。当前形态是一个可直接部署到 Vercel 的静态 HTML 站点：

- `index.html`：汇总主页面，负责展示报告入口。
- `research/mmk_sz.html`：Telegram 数据社科田野报告。
- `research/hanbalongwang_summary.html`：《憨巴龙王》小节化深度阅读报告。

所有页面都尽量保持单文件交付，不依赖后端服务，适合本地打开、静态托管和长期归档。

## 当前报告

### TG Dating Research Report

`research/mmk_sz.html` 是一份基于 Telegram 频道导出数据整理而成的可搜索、可阅读、带匿名化处理的社科风格研究报告。

它不是原始数据备份，而是把频道中的结构化字段、随机样本、文本摘抄和统计结果整合成了一份单文件 HTML 报告，适合浏览、检索和展示。

报告主要包含：

- 概览：对样本整体特征做摘要说明。
- 硬数据：展示全量记录数、抽样规模、消费中位数、评分均值等核心统计指标。
- 类型总评：对不同类型的评价模式做归纳。
- 评价语显微镜：从文本里提取高频表达、叙事习惯和评价语言。
- 亮点语句：展示有代表性的匿名化摘抄。
- 原始摘抄库：提供可折叠、可搜索的原始文本样本库，便于回查。

### 《憨巴龙王》小节化深度阅读

`research/hanbalongwang_summary.html` 是一份可搜索的单文件 HTML 阅读报告。

它把原始 PDF 按“语义小节”重新组织，而不是按页码切分。每个小节保留原页截图作为证据，同时提供内容整理、归纳总结、策略抽象、延展分析、风险提示和可继续追踪的变量。

报告主要包含：

- 小节卡片：按仓位、信息差、协议资产、KOL、广告层等主题组织。
- 原页截图：点击缩略图可放大校验原文。
- 搜索与筛选：支持按关键词、主题和风险等级定位小节。
- 主题索引：把跨页内容合并为可阅读的语义单元。
- 风险提示：区分可复用的决策框架和不可复制的营销、情绪或极端风险叙事。

## 本地查看

直接打开首页：

```bash
open index.html
```

也可以直接打开某个分页面：

```bash
open "research/mmk_sz.html"
open "research/hanbalongwang_summary.html"
```

## 新增分析页面

后续新增静态报告时，建议遵循这个流程：

1. 将新的单文件 HTML 放到 `research/` 目录，文件名尽量使用英文、数字和连字符，例如 `some-topic-report.html`。
2. 在 `index.html` 的 `#reports` 区域复制一张 `.report-card`，修改标题、标签、摘要和 `href`。
3. 如果报告还没有完成，可以先使用 `.queue-card` 作为预留入口。
4. 保持每个分页面独立可访问，避免依赖本地数据文件或后端接口。

## Vercel 部署

这是纯静态项目，不需要构建命令。

推荐 Vercel 设置：

- Framework Preset: `Other`
- Build Command: 留空
- Output Directory: `.`
- Install Command: 留空

部署完成后，Vercel 会默认访问 `index.html`，其他 HTML 分页面可以通过首页卡片进入。

## 内容原则

- 数据报告应保留研究用途说明。
- 涉及个人、地点、作者、联系方式等信息时，需要做匿名化、脱敏或遮罩。
- 首页只做汇总导航，不承载过多分析细节。
- 分页面可以自由做搜索、图表、样本库、交互筛选等具体功能。
